requireConcurrentIndexDeletion

Diagnostic Category: lint/safety/requireConcurrentIndexDeletion

Since: vnext

Sources: - Inspired from: squawk/require-concurrent-index-deletion

Description

Dropping indexes non-concurrently can lock the table for reads.

When dropping an index, using DROP INDEX without CONCURRENTLY will lock the table preventing reads and writes for the duration of the drop. This can cause downtime in production systems. Use DROP INDEX CONCURRENTLY to drop the index without blocking concurrent operations.

Examples

Invalid

DROP INDEX IF EXISTS users_email_idx;
